üìê Vectors: Your Position & Direction Tool

What is a Vector?

Think of a vector as an arrow that points from one place to another. It has two important properties:

  • Direction: Which way it's pointing
  • Length (magnitude): How far it goes

Real-World Analogy

Imagine you're telling a friend how to walk to a store. You could say:

  • "Walk 3 blocks east, then 2 blocks north"
  • Or in math: Vector = (3, 2)

The vector (3, 2) means: move 3 units in the X direction (horizontal) and 2 units in the Y direction (vertical).

üìù Click to see: Adding Movement (Step 2) 
class Vector2D {
    constructor(x, y) {
        this.x = x;
        this.y = y;
    }

    // Add another vector to this one
    add(other) {
        this.x += other.x;  // Move horizontally
        this.y += other.y;  // Move vertically
        return this;        // Allow chaining
    }
}

// Player starts at (100, 100)

// Player presses 'W' key - move up

player.add(movement);

// Now player is at (100, 95) - moved up 5 pixels!
3 Add distance calculation (for collision detection):
üìù Click to see: Measuring Distance (Step 3) 
distance(other) {
    // Use Pythagorean theorem: ‚àö(dx¬≤ + dy¬≤)
    let dx = other.x - this.x;  // Horizontal distance
    let dy = other.y - this.y;  // Vertical distance
    return Math.sqrt(dx * dx + dy * dy);
}

üîß Essential Vector Operations

1. Normalize (Make Unit Length)

Problem: You want the player to always move at the same speed, regardless of direction.

Solution: Normalize the vector (make it length = 1), then multiply by desired speed.

üìù Click to see: Normalization Code 
normalize() {
    let length = this.length();
    if (length > 0) {
        this.x /= length;  // Divide by length
        this.y /= length;
    }
    return this;
}

length() {
    return Math.sqrt(this.x * this.x + this.y * this.y);
}

üéÆ Constant Speed Movement:

// Get input direction (could be diagonal!)
let direction = new Vector2D(
    input.right - input.left,    // -1, 0, or 1
    input.down - input.up
);

// Normalize so diagonal isn't faster!
direction.normalize();

// Apply constant speed
let speed = 5;
direction.multiply(speed);

// Add to player position
player.add(direction);
üí° Why this matters: Without normalization, moving diagonally would be 1.4x faster (because of Pythagorean theorem)! This is a common bug in games.

2. Dot Product (Understanding Angles)

What it tells you: How much two vectors point in the same direction.

  • Positive: Vectors point roughly the same way
  • Zero: Vectors are perpendicular (90 degrees)
  • Negative: Vectors point opposite ways
üìù Click to see: Dot Product Code 
dot(other) {
    return this.x * other.x + this.y * other.y;
}

üéÆ Check if Enemy is in Front:

// Player is facing direction
let facingDir = new Vector2D(1, 0).normalize();

// Direction to enemy
let toEnemy = enemy.subtract(player).normalize();

// Check if in front
let dotProduct = facingDir.dot(toEnemy);

if (dotProduct > 0) {
    console.log("Enemy is in front!");
} else {
    console.log("Enemy is behind!");
}

üéÆ Building Your First Game System

Let's put it all together and create a simple game with a player, enemies, and projectiles!

Complete Game Loop

üìù Click to see: Complete Game Code 
class Player {
    constructor(x, y) {
        this.position = new Vector2D(x, y);
        this.velocity = new Vector2D(0, 0);
        this.speed = 5;
    }

    update(input) {
        // Get input direction
        let dir = new Vector2D(
            input.right - input.left,
            input.down - input.up
        );

        // Normalize and apply speed
        if (dir.length() > 0) {
            dir.normalize().multiply(this.speed);
        }

        // Update position
        this.position.add(dir);
    }

    draw(ctx) {
        ctx.fillStyle = '#4fc3f7';
        ctx.beginPath();
        ctx.arc(this.position.x, this.position.y, 20, 0, Math.PI * 2);
        ctx.fill();
    }
}

‚ù§Ô∏è Health & Damage System

Why Health Systems Matter

Health systems control when entities die, how much damage they take, and provide feedback to players. They're essential for any game with combat or hazards.

Key Components:

  • Current Health: How much life remains
  • Max Health: Maximum possible health
  • Damage: Reduce health when hit
  • Heal: Restore health (capped at max)
  • Death: What happens at 0 health
üìù Click to see: Health System Implementation 
class HealthSystem {
    constructor(maxHealth) {
        this.maxHealth = maxHealth;
        this.currentHealth = maxHealth;
        this.isAlive = true;
        this.isInvulnerable = false;
        this.invulnerabilityTimer = 0;
    }

    takeDamage(amount) {
        if (this.isInvulnerable || !this.isAlive) return false;

        this.currentHealth -= amount;
        this.currentHealth = Math.max(0, this.currentHealth);

        // Brief invulnerability after hit (i-frames)
        this.makeInvulnerable(1000); // 1 second

        if (this.currentHealth <= 0) {
            this.die();
        }

        return true; // Damage was dealt
    }

    heal(amount) {
        if (!this.isAlive) return;

        this.currentHealth += amount;
        this.currentHealth = Math.min(this.maxHealth, this.currentHealth);
    }

    makeInvulnerable(duration) {
        this.isInvulnerable = true;
        this.invulnerabilityTimer = duration;
    }

    update(deltaTime) {
        if (this.invulnerabilityTimer > 0) {
            this.invulnerabilityTimer -= deltaTime;
            if (this.invulnerabilityTimer <= 0) {
                this.isInvulnerable = false;
            }
        }
    }

    die() {
        this.isAlive = false;
        this.currentHealth = 0;
    }

    getHealthPercentage() {
        return (this.currentHealth / this.maxHealth) * 100;
    }
}

üèÜ Scoring & Combo System

Why Scoring Matters

Scoring gives players goals and rewards skill. Combo systems add depth by rewarding consecutive actions.

Key Features:

  • Base Score: Points for actions
  • Multipliers: Increase points earned
  • Combo Counter: Track consecutive hits
  • Combo Timer: Reset if too slow
  • High Score: Best performance tracking
üìù Click to see: Scoring System Implementation 
class ScoringSystem {
    constructor() {
        this.score = 0;
        this.combo = 0;
        this.comboTimer = 0;
        this.comboTimeout = 2000; // 2 seconds
        this.highScore = 0;
    }

    addScore(basePoints) {
        // Combo multiplier
        const multiplier = 1 + (this.combo * 0.1);
        const finalPoints = Math.floor(basePoints * multiplier);

        this.score += finalPoints;
        this.combo++;
        this.comboTimer = this.comboTimeout;

        // Update high score
        if (this.score > this.highScore) {
            this.highScore = this.score;
        }

        return finalPoints;
    }

    update(deltaTime) {
        if (this.comboTimer > 0) {
            this.comboTimer -= deltaTime;
            if (this.comboTimer <= 0) {
                this.breakCombo();
            }
        }
    }

    breakCombo() {
        this.combo = 0;
        this.comboTimer = 0;
    }

    reset() {
        this.score = 0;
        this.breakCombo();
    }

    getComboMultiplier() {
        return 1 + (this.combo * 0.1);
    }
}

üéí Inventory System

Managing Items

Inventory systems let players collect, store, and use items. Essential for RPGs, survival games, and more!

Core Operations:

  • Add Item: Pickup or receive items
  • Remove Item: Use or drop items
  • Stack Items: Group identical items
  • Check Capacity: Inventory limits
  • Find Item: Search inventory
üìù Click to see: Inventory System Implementation 
class Item {
    constructor(id, name, stackable = false, maxStack = 1) {
        this.id = id;
        this.name = name;
        this.stackable = stackable;
        this.maxStack = maxStack;
        this.quantity = 1;
    }
}

class Inventory {
    constructor(capacity = 20) {
        this.items = [];
        this.capacity = capacity;
    }

    addItem(item) {
        if (item.stackable) {
            // Try to stack with existing item
            const existing = this.items.find(i => i.id === item.id);
            if (existing && existing.quantity < existing.maxStack) {
                existing.quantity++;
                return true;
            }
        }

        // Add as new item if space available
        if (this.items.length < this.capacity) {
            this.items.push(item);
            return true;
        }

        return false; // Inventory full
    }

    removeItem(itemId, quantity = 1) {
        const index = this.items.findIndex(i => i.id === itemId);
        if (index === -1) return false;

        const item = this.items[index];
        item.quantity -= quantity;

        if (item.quantity <= 0) {
            this.items.splice(index, 1);
        }

        return true;
    }

    hasItem(itemId, quantity = 1) {
        const item = this.items.find(i => i.id === itemId);
        return item && item.quantity >= quantity;
    }

    getItemCount(itemId) {
        const item = this.items.find(i => i.id === itemId);
        return item ? item.quantity : 0;
    }

    isFull() {
        return this.items.length >= this.capacity;
    }
}

‚è±Ô∏è Timer & Cooldown System

Managing Time in Games

Timers control when actions can happen, create urgency, and prevent spam. They're everywhere in games!

Common Uses:

  • Cooldowns: Limit ability usage
  • Countdowns: Time limits on objectives
  • Buffs/Debuffs: Temporary effects
  • Respawn Timers: Enemy/item spawning
üìù Click to see: Timer System Implementation 
class Timer {
    constructor(duration) {
        this.duration = duration;
        this.remaining = duration;
        this.isActive = false;
        this.isPaused = false;
    }

    start() {
        this.isActive = true;
        this.remaining = this.duration;
    }

    update(deltaTime) {
        if (!this.isActive || this.isPaused) return false;

        this.remaining -= deltaTime;

        if (this.remaining <= 0) {
            this.remaining = 0;
            this.isActive = false;
            return true; // Timer finished
        }

        return false;
    }

    reset() {
        this.remaining = this.duration;
    }

    getProgress() {
        return 1 - (this.remaining / this.duration);
    }

    isReady() {
        return !this.isActive;
    }
}

class Cooldown extends Timer {
    use() {
        if (!this.isReady()) return false;

        this.start();
        return true; // Ability used successfully
    }
}

‚ö° Power-up System

Creating Collectible Power-ups

Power-ups are temporary or permanent boosts that enhance player abilities. They add variety and excitement to gameplay!

Key Components:

  • Power-up Types: Different effects (speed, shield, weapon upgrades)
  • Duration: How long the effect lasts
  • Collection: Detecting when player picks them up
  • Effects: What they modify (speed, damage, health)
  • Visual Feedback: Show active power-ups to player
üìù Click to see: Power-up System Implementation 
class PowerUp {
    constructor(type, x, y) {
        this.type = type;
        this.position = new Vector2D(x, y);
        this.radius = 15;
        this.collected = false;
        this.rotation = 0;

        // Power-up properties based on type
        this.config = this.getConfig(type);
    }

    getConfig(type) {
        const configs = {
            'speed': {
                color: '#4fc3f7',
                icon: '‚ö°',
                duration: 5000,
                effect: { speedMultiplier: 1.5 }
            },
            'shield': {
                color: '#66bb6a',
                icon: 'üõ°Ô∏è',
                duration: 8000,
                effect: { invulnerable: true }
            },
            'damage': {
                color: '#ef5350',
                icon: 'üî•',
                duration: 6000,
                effect: { damageMultiplier: 2 }
            },
            'health': {
                color: '#ffa726',
                icon: '‚ù§Ô∏è',
                duration: 0,  // Instant effect
                effect: { healAmount: 50 }
            }
        };
        return configs[type];
    }

    update(deltaTime) {
        // Rotate for visual effect
        this.rotation += deltaTime * 0.002;
    }

    checkCollision(playerPos, playerRadius) {
        const distance = this.position.distance(playerPos);
        return distance < (this.radius + playerRadius);
    }

    draw(ctx) {
        ctx.save();
        ctx.translate(this.position.x, this.position.y);
        ctx.rotate(this.rotation);

        // Draw outer glow
        ctx.fillStyle = this.config.color;
        ctx.globalAlpha = 0.3;
        ctx.beginPath();
        ctx.arc(0, 0, this.radius + 5, 0, Math.PI * 2);
        ctx.fill();

        // Draw main circle
        ctx.globalAlpha = 1;
        ctx.beginPath();
        ctx.arc(0, 0, this.radius, 0, Math.PI * 2);
        ctx.fill();

        // Draw icon
        ctx.font = '20px Arial';
        ctx.textAlign = 'center';
        ctx.textBaseline = 'middle';
        ctx.fillText(this.config.icon, 0, 0);

        ctx.restore();
    }
}

class PowerUpManager {
    constructor() {
        this.activePowerUps = [];  // Currently active on player
        this.availablePowerUps = [];  // In the world to collect
    }

    collectPowerUp(powerUp) {
        if (powerUp.config.duration > 0) {
            // Timed power-up
            this.activePowerUps.push({
                type: powerUp.type,
                config: powerUp.config,
                timeRemaining: powerUp.config.duration
            });
        } else {
            // Instant effect (like health)
            return powerUp.config.effect;
        }
    }

    update(deltaTime) {
        // Update active power-up durations
        this.activePowerUps = this.activePowerUps.filter(powerUp => {
            powerUp.timeRemaining -= deltaTime;
            return powerUp.timeRemaining > 0;
        });
    }

    hasEffect(effectName) {
        return this.activePowerUps.some(p =>
            p.config.effect[effectName] !== undefined
        );
    }

    getMultiplier(multiplierName) {
        const powerUp = this.activePowerUps.find(p =>
            p.config.effect[multiplierName] !== undefined
        );
        return powerUp ? powerUp.config.effect[multiplierName] : 1;
    }
}

üéÆ Using Power-ups in Your Game:

// Create power-up manager
const powerUpManager = new PowerUpManager();

// Spawn a speed power-up
const speedBoost = new PowerUp('speed', 200, 200);

// In game loop - check collision
if (speedBoost.checkCollision(player.position, player.radius)) {
    powerUpManager.collectPowerUp(speedBoost);
    // Remove from world
    availablePowerUps.splice(index, 1);
}

// Apply speed multiplier
const speedMultiplier = powerUpManager.getMultiplier('speedMultiplier');
player.speed = baseSpeed * speedMultiplier;
üí° Design Tip: Power-ups should be balanced - powerful enough to feel exciting, but temporary enough to not break the game. Visual and audio feedback makes collecting them feel rewarding!
